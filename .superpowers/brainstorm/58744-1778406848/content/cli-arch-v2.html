设计要点

核心设计

  • MCP Registry 启动时连接所有配置的 Server
  • 自动发现每个 Server 的 tools 列表
  • 所有 tool 统一暴露给 LLM，LLM 不感知哪个 MCP 提供的
  • Tool call 时按 tool name 路由到对应 MCP Server 执行
  • 单个 MCP Server 崩溃不影响其他

配置方式

  • .mcp.json — 同 Claude Code 格式，兼容现有 MCP 生态
  • 支持 stdio / HTTP 两种传输
  • 三个监控 MCP 只是初始配置
  • 随时加新的 MCP Server，零代码改动
